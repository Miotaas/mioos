⊡ Decide — canonical home for all decisions. The only place you Approve, Reject, or Request changes.
Layout principle: Left panel = list of all pending decisions (filterable by type). Right panel = detail view for the selected item. Actions live in the detail panel — not the list. The founder selects an item, reads full context, then decides.
Nothing lives here permanently. Every item in Decide either gets approved (→ Projects or executed), rejected (→ archived), or deferred (stays, deprioritized). Decide is a processing surface, not a storage layer.
