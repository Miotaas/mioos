MioOS V3 — Projects Structure

Three organizational models — and one question that changes the answer.

Before the models, one question that changes everything:
Do personal items (Japan, gym, financial goals) live in Projects — or in Life?

If they live in Projects: Life may not need to exist. Projects handles everything from AION to Japan.
If they live in Life: Projects stays clean as a work-only layer. Life is a distinct personal space.

This decision is addressed at the bottom after the three models.

Trade-off: Clean when every Initiative has a clear parent Venture. Gets messy when an initiative is standalone (not attached to any venture). Also forces a decision on every initiative: "which venture does this belong to?" — which is overhead the founder shouldn't have. The flat two-type model avoids this. Initiatives can optionally reference a parent Venture without being structurally nested.

The question beneath the question: where do personal projects live?

Option A: Personal projects stay in Projects

  • Japan trip, gym, financial goals live in Projects as "Personal" type
  • Pro: One place for all tracked work
  • Pro: Life page may not be needed
  • Con: AION next to gym goal feels wrong
  • Con: Personal items don't have AI teams, revenue, or approval flows — the Projects UI is built for things that do
  • Con: Life disappears — but calendar, habits, health still need a home
Life becomes redundant. Calendar and habits need to merge into Projects or Today.

Option B: Personal projects live in Life (recommended)

  • Japan trip → Life (travel)
  • Gym / health → Life (habits + health)
  • Financial goals → Life (personal goals)
  • Pro: Projects is clean, work-only
  • Pro: Life has its own character — calendar, habits, travel, personal goals all belong together
  • Pro: The separation matches how the founder actually thinks: work mode vs life mode
  • Con: A complex personal project (e.g., launching a personal brand) has no clear home
Projects = work. Life = personal. The mental model is clear and natural.

Recommended: Model B + Life split

Ventures tab
Ongoing businesses with revenue potential. AION, MioOS, Mail Co-Pilot. Short list. Always visible. These are your core commitments.
Initiatives tab
Bounded experiments. Vertical Templates, pilots, proofs-of-concept. Can reference a parent Venture optionally, but not structurally nested. Moves through Exploring → Validating → Deciding → Building → Done/Killed.
Personal → Life
Japan, gym, financial goals, personal targets — all live in Life. Projects doesn't see them. Life has a different character: calendar integration, habits, health, travel.
Edge case
A personal project that has work-like complexity (e.g., publishing a book, building a personal brand) can be an Initiative. The rule: does it involve tasks, AI teams, or go/no-go decisions? If yes → Initiative. If no → Life.
