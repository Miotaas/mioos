MioOS V3 — Information Ownership Model

Every piece of information has exactly one canonical home. Today surfaces only — it owns nothing.

Core principle: One truth, one place.
If information appears in two pages, one of them is wrong.
Today is a lens, not a database. Every signal on Today is a pointer to its canonical home.

Pages eliminated from current MioOS

Departments / Workforce
Merged into Teams. Founders don't manage org charts — they dispatch work.
Pending Actions (standalone)
Becomes Decide. Same function, clearer name, canonical ownership.
Goals (standalone page)
Business goals live inside Projects. Personal goals live inside Life. No standalone page needed.
Requests / Command Console
Work dispatch moves to Teams. No separate destination needed.
Opportunities (standalone)
Unapproved opportunities → Decide (need judgment). Approved → Projects. Never standalone.
Revenue Health (standalone)
Revenue lives inside each Project. A revenue summary surfaces on Today. No standalone page.

Final MioOS V3 navigation — 6 destinations total

☀ Today
Briefing surface. Surfaces signals from all other pages. Owns nothing.
⊡ Decide
All decisions: approvals, drafts, proposals, AI outputs. The only place decisions are processed.
◈ Projects
All ventures, tasks, goals, blockers, revenue. The only place project work lives.
◎ Life
Calendar, personal goals, health, travel, reminders. The only place personal life lives.
⚡ Teams
AI workforce. Dispatch work, view activity, configure teams. Outputs go to Decide automatically.
⚙ Settings
Configuration only. No operational content.
